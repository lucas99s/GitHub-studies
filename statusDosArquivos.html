Ciclo de vida dos Status dos Arquivos

Existem 4 estados em que um arquivo pode estar.

Untracked se refere aos arquivos novos ou que foram removidos do git que o git não reconhece.
Unmodified são os arquivos que foram comitados e não sofreram modificação após esse commit.
Modified são os arquivos que após terem sido comitados, foram alterado.
Staged está relacionado aos arquivos que estão prontos para serem comitados. Após esse commit, os arquivos passam a ter o status de unmodified

O comando para você reconhecer os status do seus arquivos é o 'git status'

@User
MINGW64
~/Git Directory
(master)
$ git status
On branch master
Changes not staged for commit:

modified: index.html
modified: iniciandoRepositorio.html
modified: lessonStyle.css
modified: style.css

Untracked files:

img/
statusDosArquivos.html

@User
MINGW64
~/Git Directory
(master)

Esses são os arquivos alterados e criados no momento em que produzo essa página, você pode notar que os primeiros 4 arquivos em vermelho, são os que já tinha sido comitados anteriormente, por isso estão marcados como modified, pois a versão que foi comitada e essa não são iguais.

Os dois últimos estão com o status de Untracked, ou seja, eles são novos e nunca foram comitados antes. O primeiro é referente a pasta nova, onde armazenarei imagens como aquela no topo, e o segundo é essa página.

Para que os arquivos possam ser comitados, primeiro necessita colocar ele 'on stage', o comando para isso é git add 'nomeDoArquivo'. Podemos fazer isso para cada um do arquivo, mas como tem mais de um arquivo que precisa ser colocado on stage, podemos usar o '*', para se referir a todos que não foram adicionados

@User
MINGW64
~/Git Directory
(master)
$ git add *
@User
MINGW64
~/Git Directory
(master)

Feito isso, tanto os arquivos com o status unmodified, quanto os untracked foram colocados on stage e estão prontos para serem comitados. Você pode conferir isso digitando 'git status'

@User
MINGW64
~/Git Directory
(master)
$ git status
On branch master
Changes to be committed:

new file: img/statusFiles.png
modified: index.html
modified: iniciandoRepositorio.html
modified: lessonStyle.css
new file: statusDosArquivos.html
modified: style.css

Note que você consegue ver que os arquivos que antes eram untracked, agora no stage estão como 'new file'.
